CSS Modules in React - Detailed Q&A

1. What are CSS Modules in React?

CSS Modules are a way to scope your CSS classes locally to a component in React, preventing any global class name conflicts. This is achieved by automatically generating unique class names for every CSS class, allowing developers to write styles that are only applied to the component they belong to.

/* MyComponent.module.css */
.myComponent {
  background-color: #f0f0f0;
  padding: 10px;
}

In your React component:

import React from 'react';
import styles from './MyComponent.module.css';

const MyComponent = () => {
  return <div className={styles.myComponent}>Hello, World!</div>
};

export default MyComponent;

2. How do you set up CSS Modules in a React project?

To set up CSS Modules in React, you need to use a bundler like Webpack. Fortunately, Create React App (CRA) has built-in support for CSS Modules, so you don’t need any extra configuration.

In your CRA project, you can simply rename your CSS file to filename.module.css and import it as a module in your component.

/* MyComponent.module.css */
.button {
  color: red;
}
    
import React from 'react';
import styles from './MyComponent.module.css';

const MyComponent = () => {
  return <button className={styles.button}>Click me</button>
};

export default MyComponent;

3. How do you import and use CSS Modules in a React component?

You import a CSS Module like any other module in JavaScript. The main difference is that CSS modules are imported as an object, with keys representing the CSS class names and values being the unique class names generated by the module bundler (e.g., Webpack).

import React from 'react';
import styles from './MyComponent.module.css';

const MyComponent = () => {
  return <div className={styles.className}>Styled Component</div>
};

export default MyComponent;

4. Can you use global CSS in a project with CSS Modules?

Yes, you can still use global CSS alongside CSS Modules. For global styles, you can include a regular CSS file and use it like usual. The difference is that global styles will apply to the entire application, while CSS Modules are scoped to their components.

/* global.css */
body {
  font-family: Arial, sans-serif;
}
    
import './global.css';
import React from 'react';
import styles from './MyComponent.module.css';

const MyComponent = () => {
  return <div className={styles.className}>Global and Local Styles</div>
};

export default MyComponent;

5. How do you use dynamic class names with CSS Modules?

CSS Modules allow you to combine dynamic class names with static ones. You can use JavaScript expressions inside the className prop to combine classes based on some logic or state.

import React, { useState } from 'react';
import styles from './Button.module.css';

const Button = () => {
  const [isActive, setIsActive] = useState(false);

  return (
    <button
      className={\`\${styles.button} \${isActive ? styles.active : ''}\`}
      onClick={() => setIsActive(!isActive)}
    >
      Toggle
    </button>
  );
};

export default Button;

6. How can you prevent class name conflicts using CSS Modules?

CSS Modules prevent class name conflicts by generating unique class names for each component. This ensures that even if two components have a class with the same name, they will be scoped to their respective components.

/* Button.module.css */
.button {
  color: blue;
}
    
/* Header.module.css */
.button {
  color: green;
}

When imported into the components:

import React from 'react';
import buttonStyles from './Button.module.css';
import headerStyles from './Header.module.css';

const MyComponent = () => {
  return (
    <div>
      <button className={buttonStyles.button}>Button</button>
      <button className={headerStyles.button}>Header Button</button>
    </div>
  );
};

export default MyComponent;

Yes, you can import multiple CSS Modules into a single React component. Each imported module will be scoped locally, preventing any name clashes. You just need to import each CSS module and reference them appropriately within the component.

9. What happens when you don’t use the module in your import?

If you import a CSS file without the .module.css extension, it will be treated as a global CSS file, and all the class names defined in it will be global. This means there could be potential conflicts with other classes in your app.

/* MyComponent.css */

  color: red;

16. Can you pass props to CSS Modules?

CSS Modules themselves don’t accept props, but you can dynamically change the class names based on component props. This can be done by conditionally rendering class names depending on the props passed to the component.

  const buttonClass = isPrimary ? styles.primary : styles.secondary;

  return <button className={buttonClass}>Click Me</button>
