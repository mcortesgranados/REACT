useState Adds state to functional components. ✨ Storing and updating primitive or complex data types, like numbers, strings, arrays. 📊 Current state value and a function to update it. The state value itself, typically used in useEffect. 🧑‍💻 When you need local state management in a functional component. 🏗️ Simple to use, intuitive API. ✅ Can lead to unnecessary re-renders if state updates are frequent. 🔄 Managing simple state, e.g., form inputs, counters, toggles. ⏱️
useEffect Handles side effects like data fetching or subscriptions. 🔄 Fetching data, updating the DOM, setting up timers, or subscribing to external events. 🌐 None (it triggers side effects). 🌪️ An array of dependencies; optional. 🔗 When you need to perform side effects after render. 🛠️ Flexible, runs after render. Great for async operations. ⏳ Can lead to race conditions if not managed properly with dependencies. ⚠️ Data fetching, event listeners, DOM manipulation. 📡
useContext Accesses the value of a React context. 🌍 Sharing data like themes or authentication status without manually passing props. 🎨 The current context value. 💡 The context object passed to useContext. 📂 When you need access to a context value within a component. 🔍 Eliminates prop drilling. Simplifies state management. 🎉 Potential performance overhead if not optimized (re-renders when context changes). 🐢 Theme management, global user authentication, settings. 🔐
useReducer Manages more complex state logic with a reducer. ⚙️ Handling complex state transitions in large applications or forms. 🏢 The current state and a dispatch function. 🚀 The reducer function and initial state (optional). 📑 When state logic becomes too complex for useState. 🏋️‍♂️ Useful for managing complex state logic. Easier to scale. 🏢 Slightly more boilerplate code compared to useState. 📚 Managing complex forms, state-driven applications. 📝
useCallback Memoizes a callback function. 🔁 Prevents re-creation of a function on each render, optimizing performance, especially in child components. ⚡ The memoized callback function. 🎯 An array of dependencies (values that the function depends on). 📅 When you need to optimize performance by preventing unnecessary function re-creations. 💨 Prevents unnecessary re-creations of functions, optimizing performance. ⚡ Can be overused or misused, potentially adding unnecessary complexity. 💭 Optimizing event handlers, preventing re-renders in child components. 🎯
useMemo Memoizes a computed value. 🔐 Optimizing expensive calculations or ensuring stable references for complex objects. 🧠 The memoized value. 💎 An array of dependencies (values that trigger recomputation). 🔄 When you want to optimize heavy computations or object references. 🏃‍♂️ Optimizes expensive calculations, reducing unnecessary computations. ⚙️ Misuse can lead to excessive memory usage or redundant computation. 💾 Optimizing performance in data-heavy apps or calculations. 📊
useRef Creates a mutable object that persists across renders. 🔄 Accessing DOM elements directly or storing mutable values like previous state values. 🌿 A mutable ref object. 🏷️ No dependencies (can be updated directly). 🔧 When you need to access or modify DOM elements or store mutable values between renders. 🏗️ Maintains persistent references across renders. Useful for DOM manipulation. 💻 No automatic re-render on value change. Can lead to stale data if not used carefully. ⏳ Storing previous values, focusing on DOM elements, or mutable values. 🌍
useImperativeHandle Customizes the instance value exposed by ref. 🛠️ Controlling a child component’s instance or exposing specific methods to parent components. 🎮 A custom object or method exposed to the parent. 👨‍👩‍👧‍👦 The ref object and optional dependencies. 🔗 When you need to expose a specific set of methods or values to a parent component through ref. 🏠 Exposes specific functionality to parent components, providing flexibility. 🌟 Can create tightly coupled components. Overuse may break encapsulation. 🏚️ Controlling child component methods, handling complex UI interactions. 🖥️
useLayoutEffect Runs synchronously after all DOM mutations. 🔄 Measuring and modifying the DOM before the browser paints, to avoid layout shifts. 🎨 None (it triggers side effects similar to useEffect). ⚡ An array of dependencies, like in useEffect. 🔁 When you need to modify the DOM before it paints or when you need to measure layout for performance reasons. 📏 Synchronous, useful for DOM manipulation before rendering. 💡 Can block the paint, potentially causing performance issues. 🐢 Layout calculations, animations, measurements before rendering. 🎥
