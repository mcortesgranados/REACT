React Topic 01_06: Conditional Rendering

1. What is conditional rendering in React?

Answer: Conditional rendering in React means showing or hiding parts of the UI based on conditions using JavaScript logic.

function Greeting(props) {
  if (props.isLoggedIn) {
    return <h1>Welcome back!</h1>;
  }
  return <h1>Please sign up.</h1>;
}

2. How do you use the ternary operator for conditional rendering?

Answer: The ternary operator is a concise way to conditionally render elements inline.

function Status(props) {
  return (
    <p>
      {props.online ? 'User is online' : 'User is offline'}
    </p>
  );
}

3. How can you conditionally render nothing?

Answer: Return null from the component or conditionally skip rendering by short-circuit evaluation.

function OptionalMessage(props) {
  if (!props.show) return null;
  return <p>This is a secret message.</p>;
}

4. What is short-circuit evaluation in React rendering?

Answer: It's using logical operators like && to render elements only if a condition is true.

function Notification(props) {
  return (
    <div>
      {props.hasMessage && <p>You have a new message!</p>}
    </div>
  );
}

5. How do you toggle content based on component state?

Answer: You can toggle content by using useState to manage the condition.

import React, { useState } from 'react';

function ToggleText() {
  const [visible, setVisible] = useState(true);

  return (
    <div>
      <button onClick={() => setVisible(!visible)}>Toggle</button>
      {visible && <p>This text is visible</p>}
    </div>
  );
}

6. Can you use if-else statements directly in JSX?

Answer: No, you can't use traditional if-else statements inside JSX. Use them outside the return statement or convert the logic into ternary operators or logical operators.

function Message(props) {
  let content;
  if (props.type === 'success') {
    content = <p>Success!</p>;
  } else {
    content = <p>Something went wrong.</p>;
  }

  return <div>{content}</div>;
}

7. How do you use an IIFE (Immediately Invoked Function Expression) inside JSX?

Answer: You can use an IIFE to execute more complex conditional logic within JSX expressions.

function Alert(props) {
  return (
    <div>
      {(() => {
        if (props.type === 'error') return <p>Error!</p>;
        if (props.type === 'warning') return <p>Warning!</p>;
        return <p>All good!</p>;
      })()}
    </div>
  );
}

8. Can you use switch statements for conditional rendering?

Answer: Yes, but you have to use them outside JSX since JSX doesnâ€™t support switch directly. Use it to assign a variable and return that variable in JSX.

function StatusIcon(props) {
  let icon;
  switch (props.status) {
    case 'loading':
      icon = <p>Loading...</p>;

12. Can a component return null?

Answer: Yes, a component can return null to prevent rendering anything to the DOM. This is useful for conditional rendering.

function HiddenMessage(props) {
  if (!props.show) {
    return null;
  }

  return <p>This message is visible</p>;
}

15. How does short-circuit evaluation help in conditional rendering?

Answer: Short-circuit evaluation with && allows you to render components only when a condition is true, skipping evaluation if false.

function Notifications(props) {

18. Can you render a fallback UI when a component fails to render?

Answer: Yes, using error boundaries (class components only) you can display fallback UI on errors.

class ErrorBoundary extends React.Component {
  constructor(props) {
    super(props);
    this.state = { hasError: false };
  }

  static getDerivedStateFromError(error) {
    return { hasError: true };
  }

  render() {
    if (this.state.hasError) {
      return <h1>Something went wrong.</h1>;
    }
    return this.props.children;
  }
}

19. How do you conditionally render JSX based on API response?

Answer: You manage loading and error states and render content based on the status of your fetch request.

function DataDisplay() {
  const [data, setData] = React.useState(null);
  const [loading, setLoading] = React.useState(true);

  React.useEffect(() => {
    fetch('/api/data')
      .then(res => res.json())
      .then(json => {
        setData(json);
        setLoading(false);
      });
  }, []);

  if (loading) return <p>Loading...</p>;
  return <div>{data.message}</div>;
}

24. How do you handle conditional rendering with forms in React?

Answer: Conditional rendering can be used in forms to display validation messages, error feedback, or different form layouts based on the state or input values.

function LoginForm(props) {
  const [email, setEmail] = React.useState('');
  const [error, setError] = React.useState('');

  function handleSubmit(event) {
    event.preventDefault();
    if (!email) {
      setError('Email is required');
    } else {
      setError('');
      // Submit logic here
    }
  }

  return (
    <form onSubmit={handleSubmit}>
      <input
        type="email"
        value={email}
        onChange={(e) => setEmail(e.target.value)}
      />
      {error && <p style={color: 'red'}>{error}</p>}
      <button type="submit">Submit</button>
    </form>
  );
}

25. How can you conditionally render a component based on screen size?

Answer: You can use the window.innerWidth or a library like react-responsive to conditionally render components based on screen size.

import React, { useState, useEffect } from 'react';

function ResponsiveComponent() {
  const [isMobile, setIsMobile] = useState(false);

  useEffect(() => {
    const checkSize = () => {
      setIsMobile(window.innerWidth < 768);
    };

    window.addEventListener('resize', checkSize);
    checkSize(); // Initial check

    return () => {
      window.removeEventListener('resize', checkSize);
    };
  }, []);

  return (
    {isMobile ? (
      <p>You are on a mobile device</p>
    ) : (
      <p>You are on a desktop device</p>
    )}
  );
}
