25 React Component Questions: Functional & Class-based

1. What is a functional component in React?

Answer: A functional component is a JavaScript function that returns JSX. It doesn't manage its own state unless using Hooks.

function Greeting(props) {
  return <h1>Hello, {props.name}!</h1>;
}

2. What is a class-based component in React?

Answer: A class-based component extends React.Component and can hold its own state and lifecycle methods.

class Greeting extends React.Component {
  render() {
    return <h1>Hello, {this.props.name}!</h1>;
  }
}

3. How do you add state to a functional component?

Answer: You can use the useState Hook to add state to functional components.

import React, { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

4. How do you manage state in a class component?

Answer: Use this.state to initialize and this.setState() to update the component's state.

class Counter extends React.Component {
  constructor(props) {
    super(props);
    this.state = { count: 0 };
  }

  increment = () => {
    this.setState({ count: this.state.count + 1 });
  }

  render() {
    return (
      <div>
        <p>Count: {this.state.count}</p>
        <button onClick={this.increment}>Increment</button>
      </div>
    );
  }
}

5. What are the main differences between functional and class components?

Answer: Functional components are simpler, use Hooks for state/lifecycle, and have no this keyword. Class components use this, have lifecycle methods, and are more verbose.

// Functional: concise, uses Hooks
function Example() {
  const [value, setValue] = useState(0);
  return <div>{value}</div>;
}

// Class-based: uses state and lifecycle
class Example extends React.Component {
  constructor(props) {
    super(props);
    this.state = { value: 0 };
  }
  render() {
    return <div>{this.state.value}</div>;
  }
}

6. Can functional components have lifecycle methods?

Answer: Not directly, but you can mimic lifecycle behavior using the useEffect Hook.

import React, { useEffect } from 'react';

function MyComponent() {
  useEffect(() => {
    console.log('Component mounted');
    return () => console.log('Component unmounted');
  }, []);

  return <div>Hello</div>;
}

7. What is the constructor used for in class components?

Answer: The constructor initializes the component's state and binds event handlers.

class Welcome extends React.Component {
  constructor(props) {
    super(props);
    this.state = { message: 'Hello' };
  }

  render() {
    return <h1>{this.state.message}</h1>;
  }
}

8. How do you pass props to a component?

Answer: Props are passed to components like attributes in JSX and accessed via props in functional components or this.props in class components.

// Parent
<Greeting name="Alice" />

// Functional
function Greeting(props) {
  return <h1>Hello, {props.name}!</h1>;
}

// Class-based
class Greeting extends React.Component {
  render() {
    return <h1>Hello, {this.props.name}!</h1>;
  }
}

9. How do you bind event handlers in class components?

Answer: Bind methods in the constructor or use arrow functions to ensure this refers to the correct context.

class Clicker extends React.Component {
  constructor(props) {
    super(props);
    this.handleClick = this.handleClick.bind(this);
  }

  handleClick() {
    alert('Clicked!');
  }

  render() {
    return <button onClick={this.handleClick}>Click me</button>;
  }
}

10. How do you handle events in functional components?

Answer: Use arrow functions directly or define functions inside the component body.

function Clicker() {
  const handleClick = () => {
    alert('Clicked!');
  };

  return <button onClick={handleClick}>Click me</button>;
}

11. What is the role of render() in class components?

Answer: The render() method returns the JSX that defines the UI of the component.

class Hello extends React.Component {
  render() {
    return <h1>Hello World</h1>;
  }
}

12. What are Hooks in React?

Answer: Hooks are functions that let you use state and lifecycle features in functional components (e.g., useState, useEffect).

import React, { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);
  return <div>{count}</div>;
}

13. How can you conditionally render components?

Answer: Use conditional operators (e.g., if, ternary, &&) to decide what to render.

function Greeting(props) {
  const isLoggedIn = props.isLoggedIn;
  return (
    <div>
      {isLoggedIn ? <h1>Welcome Back</h1> : <h1>Please Sign In</h1>}
    </div>
  );
}

14. How do you lift state up between components?

Answer: Move the shared state to a common parent and pass it down via props to child components.

function Parent() {
  const [data, setData] = useState('');
  return (
    <>
      <Input value={data} setValue={setData} />
      <p>Value: {data}</p>
    </>
  );
}

function Input({ value, setValue }) {
  return <input value={value} onChange={e => setValue(e.target.value)} />;
}

15. How do you use default props in a component?

Answer: You can define default values for props either using default parameters (for functional components) or defaultProps for class components.

// Functional component
function Welcome({ name = 'Guest' }) {
  return <h1>Hello, {name}!</h1>;
}

// Class-based component
typicalExample.defaultProps = {
  name: 'Guest'
};

class typicalExample extends React.Component {
  render() {
    return <h1>Hello, {this.props.name}!</h1>;
  }
}

16. What are lifecycle methods in class components?

Answer: Lifecycle methods are special methods in class components that run at different points in a component’s life, like componentDidMount, componentDidUpdate, and componentWillUnmount.

class Timer extends React.Component {
  componentDidMount() {
    this.timer = setInterval(() => console.log('tick'), 1000);
  }
  componentWillUnmount() {
    clearInterval(this.timer);
  }
  render() {
    return <p>Timer is running...</p>;
  }
}

17. What does useEffect do in functional components?

Answer: useEffect lets you perform side effects like fetching data, subscriptions, or manually changing the DOM in functional components.

import React, { useEffect } from 'react';

function Logger() {
  useEffect(() => {
    console.log('Component mounted');
    return () => console.log('Component unmounted');
  }, []);
  return <p>Check the console</p>;
}

18. Can functional components have state?

Answer: Yes, with Hooks like useState, functional components can manage their own state.

import React, { useState } from 'react';

function Clicker() {
  const [count, setCount] = useState(0);
  return <button onClick={() => setCount(count + 1)}>Clicked {count} times</button>;
}

19. How do you pass props to class components?

Answer: Props are passed to class components just like functional ones, and accessed using this.props.

function App() {
  return <Greeting name="Manuela" />;
}

class Greeting extends React.Component {
  render() {
    return <h1>Hello, {this.props.name}!</h1>;
  }
}

20. How do you use props in functional components?

Answer: You receive props as a parameter and access them directly by name inside the component body.

function Greeting(props) {
  return <h1>Hello, {props.name}!</h1>;
}

// or use destructuring
function Greeting({ name }) {
  return <h1>Hello, {name}!</h1>;
}

21. Can class components use hooks?

Answer: No, hooks are exclusive to functional components. Class components rely on lifecycle methods and instance properties.

// ❌ Not valid
class Example extends React.Component {
  const [count, setCount] = useState(0); // Invalid in class component
}

22. How do you bind event handlers in class components?

Answer: You bind them in the constructor or use class fields syntax to ensure this context is preserved.

class Clicker extends React.Component {
  constructor(props) {
    super(props);
    this.handleClick = this.handleClick.bind(this);
  }
  handleClick() {
    alert('Clicked!');
  }
  render() {
    return <button onClick={this.handleClick}>Click me</button>;
  }
}

23. How can you use default props in functional components?

Answer: Use either default parameters in the function signature or set Component.defaultProps.

function Welcome({ name = "Guest" }) {
  return <h1>Welcome, {name}!</h1>;
}

24. What are pure components in React?

Answer: A PureComponent in React is like a regular component but with shallow prop and state comparison to prevent unnecessary re-renders.

import React from 'react';

class MyPureComponent extends React.PureComponent {
  render() {
    return <h1>Pure Component: {this.props.message}</h1>;
  }
}

25. How do you create reusable components?

Answer: By designing components to receive props and separating logic from presentation, you can reuse components across the app.

function Button({ label, onClick }) {
  return <button onClick={onClick}>{label}</button>;
}
